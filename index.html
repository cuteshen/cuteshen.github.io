node-hapi接口插件hapi-swagger

发表于 2016-10-22   |   分类于 nodejs   |  

swagger的作用主要是服务启动后，它将在项目中自行记录API接口，并生成html页面展示你的接口的文档

因为最近一直在搞hapi，所以就尝试了一波，另外express，koa都有相关swagger插件

npm install hapi-swagger --save-dev

如果要从API查看文档，您还需要安装支持模板和静态内容服务的inert和vision插件。

npm install inert --save-dev
npm install vision --save-dev

在server.js中添加配置

const Hapi = require('hapi');
const Inert = require('inert');
const Vision = require('vision');
const HapiSwagger = require('hapi-swagger');
const Pack = require('./package');

const server = new Hapi.Server();
server.connection({
        host: 'localhost',
        port: 3000
});

const options = {
info: {
        'title': 'Test API Documentation',
        'version': Pack.version,
    }
};

server.register([
Inert,
Vision,
{
    'register': HapiSwagger,
    'options': options
}], (err) => {
    server.start( (err) => {
       if (err) {
            console.log(err);
        } else {
            console.log('Server running at:', server.info.uri);
        }
    });
});

服务配置之后，添加个路由

server.route({

                id : Joi.number()//npm install joi

            }
        }
    },

HapiJS是一个开源的、基于Node.js的应用框架，它适用于构建应用程序和服务，其设计目标是让开发者把精力集中于开发可重用的应用程序的业务逻辑，向开发者提供构建应用程序业务逻辑所需的基础设施。

HapiJS安装和项目配置

npm install hapi -g

配置项目

npm init
npm install --save hapi

此时，项目开发所需的一切都已经准备完毕

创建服务器

// server.js
var Hapi = require('hapi');
var server = new Hapi.Server();

server.connection({
    host: 'localhost',
    port: 8000
});

server.start(function(){
     console.log('Server running at: ', server.info.uri);
});

首先，我们需要Hapi库。
其次，我们创建一个新hapi服务器对象，并把要监听的端口号传递给服务器对象。
最后，服务器对象启动，并输出日志信息。
要说明一点，我们创建服务器对象时，可以提供主机名、IP地址，甚至是Unix的socket文件，或者是Windows系统绑定到服务器命名的管道。

路由逻辑

// server.js
var Hapi = require('hapi');
var server = new Hapi.Server(3000);

server.route({
    method: 'GET',
    path: '/',
    handler: function (request, reply) {
        reply('Hello, world!');
    }
});

server.route({
    method: 'GET',
    path: '/{name}',
    handler: function (request, reply) {
        reply('Hello, ' + encodeURIComponent(request.params.name) + "!");
    }
});

server.start(function () {
    console.log('Server running at: ', server.info.uri);
});

SonarQube的安装、配置与使用

发表于 2016-10-07   |   分类于 nodejs   |  

SonarQube是管理代码质量一个开放平台，可以快速的定位代码中潜在的或者明显的错误，下面将会介绍一下这个工具的安装、配置以及使用。

准备工作

1、jdk(java)

2、sonarqube：http://www.sonarqube.org/downloads/

3、sonar-scanner:https://sonarsource.bintray.com/Distribution/sonar-scanner-cli/sonar-scanner-2.5.zip

4、mysql

安装

1,下载好sonarqube后，解压打开bin目录，启动相应OS目录下的StartSonar，如果是win64系统，则打开使用的是win的64位系统，则打开

D:\myjob\sonar\sonarqube-5.6.3\sonarqube-5.6.3\bin\windows-x86-64\StartSonar.bat

2,启动浏览器，访问http://localhost:9000，如出现下图则表示安装成功。

配置

1,打开mysql，新建一个数据库sonar

2,打开sonarqube安装目录下的

D:\myjob\sonar\sonarqube-5.6.3\sonarqube-5.6.3\conf\sonar.properties

3,在mysql5.X节点下输入以下信息

sonar.jdbc.url=jdbc:mysql://localhost:3306/sonar?useUnicode=true&characterEncoding=utf8&rewriteBatchedStatements=true&useConfigs=maxPerformance
sonar.jdbc.username=root
sonar.jdbc.password=root
sonar.sorceEncoding=UTF-8
sonar.login=admin
sonar.password=admin

url是数据库连接地址，username是数据库用户名，jdbc.password是数据库密码，login是sonarqube的登录名，sonar.password是sonarqube的密码

4,重启sonarqube服务，再次访问http://localhost:9000，会稍微有点慢，因为要初始化数据库信息

5,数据库初始化成功后，登录

6,按照下图的点击顺序，进入插件安装页面

7,搜索chinese Pack，安装中文语言包

8,安装成功后，重启sonarqube服务，再次访问http://localhost:9000/，即可看到中文界面

使用

1,打开刚刚下载的sonar-scanner文件夹找到

D:\myjob\sonar\sonar-scanner-2.5\sonar-scanner-2.5\conf\sonar-runner.properties

2,mysql节点下输入以下信息

sonar.jdbc.url=jdbc:mysql://localhost:3306/qjfsonar?useUnicode=true&characterEncoding=utf8&rewriteBatchedStatements=true&useConfigs=maxPerformance
sonar.jdbc.username=root
sonar.jdbc.password=root

注意：如果测试项目与服务器不在同一台机子，则需要添加服务器的IP：

#----- Default SonarQube server
sonar.host.url=http://XXX.XXX.XXX.XXX:9000

3,配置环境变量

将sonar-scanner目录中的bin添加至环境变量

D:\myjob\sonar\sonar-scanner-2.5\sonar-scanner-2.5\bin

配置完成后打开cmd输入sonar-runner -version验证是否添加成功

4,打开要进行代码分析的项目根目录，新建sonar-project.properties文件

5，输入以下信息

# must be unique in a given SonarQube instance
sonar.projectKey=my:project
# this is the name displayed in the SonarQube UI
sonar.projectName=productName
sonar.projectVersion=1.0

# Path is relative to the sonar-project.properties file. Replace "\" by "/" on Windows.
# Since SonarQube 4.2, this property is optional if sonar.modules is set.
# If not set, SonarQube starts looking for source code from the directory containing
# the sonar-project.properties file.
sonar.sources=D:/product

# Encoding of the source code. Default is default system encoding
#sonar.sourceEncoding=UTF-8

其中：projectName是项目名字，sources是源文件所在的目录,目录中的‘/’不要写成‘\’

6,设置成功后，启动sonarqube服务，并启动cmd

7,在cmd进入项目所在的根目录，输入命令：sonar-runner，分析成功后会出现下图

8,打开http://localhost:9000/，我们会看到主页出现了分析项目的概要图

9,点击项目，选择问题链接，会看到分析代码的bug，哇，好多

预知更多应用，请登录官网自行学习

概念

ES6 原生提供了 Promise 对象。

js中异步编程主要指的是setTimout/setInterval、DOM事件机制、ajax，通过传入回调函数实现控制反转。主要解决问题如：callback嵌套太深代码可读性太差，并行逻辑必须串行执行。

直接打印出来看看吧，console.dir(Promise)。

Promise是一个构造函数，自己身上有all、reject、resolve这几个眼熟的方法，原型上有then、catch等同样很眼熟的方法。这么说用Promise new出来的对象肯定就有then、catch方法

new一个promise

Promise的构造函数接收一个参数，是函数，并且传入两个参数：resolve，reject，分别表示异步操作执行成功后的回调函数和异步操作执行失败后的回调函数。按照标准来讲，resolve是将Promise的状态置为fullfiled，reject是将Promise的状态置为rejected。

node-express的cookie，session，fs文件操作

发表于 2016-09-12   |   分类于 nodejs   |  

cookie数据

解析客户端cookie数据

node原始方法：

req.headers['cookie']
//a=1; b=2; c=3

express方法：

1，引入模块cookie-parser(需下载)

const cookieParser=require('cookie-parser');

2，调用

app.use(cookieParser());

3,写入cookie

res.setHeader('set-cookie','a=1')  //原生node
res.cookie(name,value)  //express

设置过期时间:

res.setHeader('set-cookie','a=1;expires=oDate') //原生node

res.cookie(name,value,options)  //express

    ----option:{maxAge:毫秒，path:'/'}

session数据

1，引入模块cookie-session(需下载)

const cookieSession=require('cookie-session');

2，调用：

app.use(cookieSession({
    name:'zns_sessid',    //session名称
    keys:['aaa','bbb','cccc'],  //循环签名session
    maxAge:20*60*1000    //过期时间
}));

app.get('/',function(req,res){
    if(req.session.visite){
        req.session.visite++;
    }else{
        req.session.visite=1;
    }
    res.send('到访了'+req.session.visite+'次');
});

也可以通过express-session模块

读取文件 fs.readFile

设置header

res.setHeader('content-type','text/html'); //node原生方法

res.set('content-type','text/html');    //express提供

设置状态码：

res.status(404)
或
res.statusCode=404;

也可以这样写

res.status(404).send(data)

完整代码：

替换readFile

读取文件在express中可直接通过res.sendFile(path)，其中path路径必须是绝对的：

res.sendFile(__dirname+'/download.html');

或

res.sendFile('download.html',{root:__dirname});

文件下载

如果原生node方式需要设置header才可以下载文件

res.setHeader('content-disposition','attachment;filename=xxxx');

通过express可直接用res.download()，无需任何设置

res.download(filename,newName)

const express=require('express');

var app=express(); //创建一个服务器

app.get(地址,fnCb)

在express中res.send()代替了原生node的res.write(),res.end()，

因此可统一使用res.send();

获取get数据

假设我的url请求是：localhost:8081/user?username=admin&password=admin

通过req.query获取的数据为{ username: 'admin', password: 'admin' }

post请求方法

若用node原始获取post数据方式是这样：

通过body-parser获取post数据

安装模块

const bodyParser=require('body-parser');

调用中间件

app.use(bodyParser.urlencoded({extended:false}))

extended:false

通过body-parser返回的对象是一个键值对，当extended为false的时候，键值对中的值就为’String’或’Array’形式，为true的时候，则可为任何数据类型。

limit

post数据大小限制，默认100
